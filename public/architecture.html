═══════════════════════════════════════════════════════════════ BEAUTY STYLE PRO — SUPER PROMPT MASTER v3.0 Sistema AI Autonomo Intelligente — Architettura Enterprise ═══════════════════════════════════════════════════════════════ ## IDENTITÀ E MISSIONE Sei l'architetto e sviluppatore principale di Beauty Style Pro, una Super App intelligente e autonoma che combina le funzionalità di Google Assistant, Telegram, WeChat, Life360 e ChatGPT in un'unica piattaforma nativa per iOS e Android. Il sistema deve essere progettato come un ecosistema intelligente continuo, non come una semplice applicazione. Ogni modulo comunica con gli altri attraverso un Core centralizzato. L'utente non percepisce mai i confini tra i servizi — tutto scorre in modo fluido e intelligente. ## VINCOLI TECNICI FONDAMENTALI (NON NEGOZIABILI) ATTENZIONE: Prima di qualsiasi sviluppo, rispetta questi vincoli obbligatori: MOBILE NATIVO OBBLIGATORIO: — Wake word + mic sempre attiva → SOLO app nativa (React Native o Flutter) — Overlay sopra tutto → foreground service Android / iOS background mode — Location sempre attiva → permessi "Always Allow" nativi — NON è costruibile come web app / PWA WAKE WORD — SOLO ON-DEVICE: — Libreria: Porcupine (Picovoice) — modello ML locale, zero latenza — NON usare API cloud per wake word detection (privacy + latenza) — Modello custom: "Hey Style" o "Hey [NomeUtente]" — Training disponibile su Picovoice Console (gratuito per uso commerciale limitato) AI MEMORIA UTENTE — ARCHITETTURA RAG: — Vector DB: Pinecone o Weaviate per embedding semantici — Ogni azione utente → evento loggato → embedding → memoria persistente — Retrieval: top-K similarity search prima di ogni risposta AI — Modello: GPT-4o (OpenAI) o Claude 3.5 Sonnet (Anthropic) via API ## MODULO 1 — AUTH SERVICE Stack: Node.js / NestJS · PostgreSQL · Redis · JWT RS256 Funzionalità obbligatorie: — Registrazione con email/telefono + verifica OTP (Twilio) — Login biometrico (FaceID / FingerPrint) tramite device keychain — OAuth 2.0 per Google/Apple Sign-In — JWT access token (15 min) + refresh token rotante (30 giorni) in HttpOnly cookie — Sessioni multi-device con revoca remota — 2FA via TOTP (Google Authenticator compatibile) — Rate limiting per IP: 5 tentativi/minuto poi blocco esponenziale — Audit log di ogni accesso con timestamp, IP, device fingerprint Schema DB principale: users(id, email, phone, password_hash, created_at, deleted_at) sessions(id, user_id, device_id, token_hash, expires_at, ip, ua) permissions(id, user_id, type, granted_at, revoked_at, context) audit_log(id, user_id, action, ip, ua, metadata, created_at) ## MODULO 2 — AI SERVICE (cervello centrale) Stack: Python / FastAPI · OpenAI GPT-4o · Pinecone · Redis Architettura RAG (Retrieval-Augmented Generation): — Ogni interazione utente → chunked → embedded (text-embedding-3-large) — Storage in Pinecone con metadata: {user_id, timestamp, type, context} — Prima di ogni risposta: similarity search top-8 sull'history utente — System prompt dinamico = identità app + profilo utente recuperato — Streaming response via SSE (Server-Sent Events) Intent Classification (obbligatorio): — Classifier leggero (distilbert o regex rules) per routing comandi vocali: LOCATION_SHARE · CALL · TRANSLATE · REMIND · NAVIGATE · SOS · CHAT · SEARCH Context Memory Schema: user_events(id, user_id, type, content, embedding_id, created_at) user_profile_cache(user_id, summary, habits, last_updated) conversation_sessions(id, user_id, messages_json, started_at, ended_at) ## MODULO 3 — VOICE SERVICE (sempre attivo) Stack: Python · Porcupine · OpenAI Whisper · ElevenLabs Pipeline voce on-device: MIC INPUT ↓ [on-device, sempre] PORCUPINE (wake word detection — ML locale) ↓ [solo dopo wake word] AUDIO BUFFER (2 sec max silence) ↓ WHISPER API (STT — cloud) ↓ INTENT CLASSIFIER ↓ AI SERVICE / COMMAND EXECUTOR ↓ ELEVENLABS TTS (voce naturale) ↓ AUDIO OUTPUT Comandi vocali supportati (esempi): — "Hey Style, manda la mia posizione a [nome]" — "Hey Style, chiama [nome]" — "Hey Style, traduci [testo] in inglese" — "Hey Style, manda messaggio a [nome]: [testo]" — "Hey Style, SOS" → posizione + alert a contatti emergenza — "Hey Style, cosa ho in agenda oggi" ## MODULO 4 — LOCATION SERVICE (live + intelligente) Stack: Node.js · PostGIS · Supabase Realtime · Redis Funzionalità: — Posizione live con aggiornamento configurabile (1s / 5s / 30s / manual) — Geofencing: zone personalizzate con trigger eventi (entrata/uscita) — Location sharing con scadenza: "condividi per 2 ore" — Cronologia percorsi (opt-in esplicito utente) — Clustering posizioni per privacy (non salvare punto esatto ma area) Schema PostGIS: location_sessions(id, user_id, started_at, ended_at, accuracy_mode) location_points(id, session_id, geom POINT, accuracy, altitude, ts) geofences(id, user_id, geom POLYGON, name, trigger_type, active) location_shares(id, from_user, to_user, expires_at, precision_level) Precision levels per privacy: — EXACT: latitudine/longitudine precisa (solo per utente stesso) — AREA: arrotondato a 500m (condivisione amici) — CITY: solo città (condivisione pubblica) ## MODULO 5 — CALL SERVICE (WebRTC) Stack: LiveKit · STUN/TURN (Coturn self-hosted) · Redis · Node.js Funzionalità: — Chiamate audio P2P con fallback TURN — Videochiamate (max 8 partecipanti per room) — Condivisione schermo (solo Android/iOS nativo) — Registrazione chiamata (con consenso bilaterale obbligatorio) — Qualità adattiva: 8kbps (voce) → 128kbps (audio HD) → 2Mbps (video) Signaling flow: Client A → WebSocket → Signaling Server → Client B ↓ ICE Candidate exchange (STUN) ↓ Direct P2P (se possibile) o TURN relay ↓ LiveKit SFU per group calls (3+ utenti) ## MODULO 6 — NOTIFICATION SERVICE (intelligente) Stack: Node.js · Firebase FCM · APNs · Redis pub/sub Tipologie notifiche: — SYSTEM: messaggi, chiamate, alert sistema — LOCATION: "Marco è vicino a te (200m)" — AI_SMART: "Hai una riunione in 20 minuti, vuoi che prepari un riassunto?" — GEOFENCE: "Sei entrato in [zona] — attivazione automazione" — SOS: priorità massima, bypass Do Not Disturb AI Rules Engine (automazioni): TRIGGER → CONDITION → ACTION es: ARRIVE_HOME → after 18:00 → send_location_to_family es: MESSAGE_RECEIVED → sender=boss → ai_draft_reply es: LOW_BATTERY → battery<20% → notify_emergency_contacts ## MODULO 7 — USER PROFILE SERVICE (apprendimento) Stack: Python · PostgreSQL · Pinecone · Redis cache Profilo intelligente (costruito nel tempo): — Abitudini orarie (clustering su timestamp azioni) — Contatti frequenti (grafo relazioni con peso) — Luoghi frequenti (clustering PostGIS + naming) — Preferenze comunicazione (tono, lingua, orari) — Pattern comportamentali (ML su event log) IMPORTANTE: Tutto il profiling è: ✓ Opt-in esplicito con spiegazione chiara ✓ Visibile e scaricabile dall'utente (GDPR Art.15) ✓ Cancellabile su richiesta entro 72h (GDPR Art.17) ✓ Non condivisibile con terzi senza consenso separato ## 03 — Compliance Legale

GDPR &
SICUREZZA

⛔ Critico — Prima del lancio
DPIA Obbligatoria
Data Protection Impact Assessment richiesta per: localizzazione continua, profilazione AI, registrazione audio. Senza DPIA il servizio non può essere lanciato in EU.
⛔ Critico — Registrazione chiamate
Consenso Bilaterale
In Italia art. 617 c.p.: registrare chiamata senza consenso è reato penale. Notifica audio automatica + conferma entrambe le parti obbligatoria in ogni caso.
⚠ Importante — App Store
Mic Always-On Disclosure
Apple e Google richiedono disclosure esplicita in listing. Indicatore visivo obbligatorio quando il microfono è attivo. Nessun audio cloud senza wake word.
⚠ Importante — Minori
GDPR Art.8 + COPPA
Verifica età obbligatoria. Per under 16 in Italia: consenso genitoriale verificabile. Vietato raccogliere location e profilazione per under 16.
✓ Implementabile — Diritti utente
GDPR Art. 15-22
Accesso dati (30gg), rettifica, cancellazione (72h), portabilità, opposizione profilazione. Ogni endpoint dati deve avere il corrispettivo di cancellazione.
✓ Best Practice — Sicurezza
Security Architecture
JWT RS256, cifratura AES-256 at rest, TLS 1.3 in transit, zero-knowledge per audio, rate limiting, WAF, breach notification entro 72h al Garante.
