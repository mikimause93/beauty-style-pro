# Deployment Guide

## Prerequisites

- Node.js 18+
- npm 9+
- Supabase CLI
- Vercel CLI (optional)
- Git

## Environments

| Environment | Branch      | URL                                |
| ----------- | ----------- | ---------------------------------- |
| Production  | `main`      | https://beautystylepro.com         |
| Staging     | `develop`   | https://staging.beautystylepro.com |
| Preview     | PR branches | Auto-generated by Vercel           |

## Frontend Deployment (Vercel)

### Setup

1. Install Vercel CLI: `npm i -g vercel`
2. Login: `vercel login`
3. Link project: `vercel link`

### Environment Variables (Vercel Dashboard)

Add these in the Vercel project settings:

```
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
VITE_STRIPE_PUBLISHABLE_KEY=pk_live_xxx
VITE_GOOGLE_MAPS_API_KEY=AIzaSyXXX
VITE_SENTRY_DSN=https://xxx@sentry.io/yyy
```

### Manual Deploy

```bash
npm run build
vercel --prod
```

### Automated Deploy (GitHub Actions)

Push to `main` triggers the `deploy.yml` workflow. Required GitHub Secrets:

- `VERCEL_TOKEN`
- `VERCEL_ORG_ID`
- `VERCEL_PROJECT_ID`

## Backend Deployment (Supabase)

### Database Migrations

```bash
# Apply all pending migrations
supabase db push --linked

# Check migration status
supabase db diff
```

### Edge Functions

```bash
# Deploy all functions
supabase functions deploy

# Deploy single function
supabase functions deploy generate-ai-preview

# Set secrets
supabase secrets set OPENAI_API_KEY=sk-xxx
supabase secrets set STABILITY_API_KEY=sk-xxx
supabase secrets set STRIPE_SECRET=sk_live_xxx
```

### Required Supabase Secrets

```
OPENAI_API_KEY
STABILITY_API_KEY
STRIPE_SECRET
SUPABASE_SERVICE_ROLE_KEY (auto-set)
SUPABASE_URL (auto-set)
```

## Mobile Apps

### iOS

```bash
npm run build
npx cap copy ios
npx cap open ios
# Then build & archive in Xcode
```

### Android

```bash
npm run build
npx cap copy android
npx cap open android
# Then build release APK/AAB in Android Studio
```

## CI/CD Pipeline

The GitHub Actions workflows handle:

1. **CI** (`ci.yml`): Lint → Build → Unit Tests → E2E Tests → Coverage Upload
2. **Deploy** (`deploy.yml`): Build → Vercel Deploy → Supabase Migrations
3. **Security** (`security-scan.yml`): npm audit weekly

See `.github/workflows/` for full details.
